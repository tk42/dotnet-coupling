```
dotnet-coupling.exe <solution.sln> [オプション]
```

## オプション

| オプション | 説明 |
|---|---|
| `--summary` | 総合サマリを表示（既定） |
| `--hotspots` | ホットスポット（高リスク結合）一覧 |
| `--violations` | ルール違反一覧（レイヤー / 循環 / 具象） |
| `--verbose` | サマリ＋ホットスポット＋違反を一括表示 |
| `--top <N>` | ホットスポットの表示件数を N 件に制限 |
| `--json <path>` | 全情報を JSON で出力 |
| `--markdown <path>` | Markdown レポートを出力 |
| `--check` | 品質ゲート判定（終了コードで合否） |
| `--min-grade <A..F>` | `--check` の合格ライン（既定: B） |

## 出力形式

<Tabs>
  <Tab title="コンソール">

```powershell
.\dist\dotnet-coupling.exe MyApp.sln --summary
```

```powershell
.\dist\dotnet-coupling.exe MyApp.sln --hotspots --top 3
```

  <Tab title="JSON">

```powershell
.\dist\dotnet-coupling.exe MyApp.sln --json report.json
```

```json
    "tool": "dotnet-coupling",
    "version": "1.0.0",
    "analysisMode": "Semantic",
    "confidence": "High",
    "solution": "MyApp.sln",
    "generatedAt": "2026-07-18T21:00:00+09:00",
    "warnings": []
  },
  "summary": {
    "score": 68,
    "grade": "C",
    "projects": 4,
    "types": 37,
    "edges": 122,
    "hotspots": 3,

  },
  "hotspots": [

```

<Callout type="note" title="省略について">
`nodes` / `edges` / `rules` 配列はここでは省略しています。実際の出力にはすべての要素が含まれます。
</Callout>

## 終了コード

| Code | 意味 |
|---:|---|
| 0 | 合格 |
| 1 | ゲート不合格 |
| 2 | 設定エラー |
| 3 | 解析失敗 |
| 9 | 内部エラー |

<Callout type="tip" title="CI での利用">
`--check --min-grade B` は、グレードが B 未満のとき exit code 1 を返します。CI のステップでこの終了コードを判定に使うことで、アーキテクチャ劣化を品質ゲートとしてブロックできます。
</Callout>

## CI 品質ゲートの例

```powershell
.\dist\dotnet-coupling.exe MyApp.sln --check --min-grade B
if ($LASTEXITCODE -ne 0) { throw "アーキテクチャ品質ゲート不合格 (grade < B)" }
```
